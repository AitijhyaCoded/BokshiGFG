# Bokshi AI: Truth in the Age of AI Illusion

**Bokshi AI** is a multi-layered consensus engine designed to verify the authenticity of news articles, social media posts, and documents. By leveraging multimodal AI, the platform cross-references claims against established databases and real-time news feeds to identify synthetically generated content and misinformation.


## 🚀 Key Features

* **Multimodal Verification**: Supports text snippets, direct URLs, and document uploads including PDFs and images.
* **Agentic Analysis Pipeline**: A three-stage verification process including claim extraction, web-scale searching across academic journals and news outlets, and evidence cross-referencing.
* **Advanced Forensics**: Utilizes neural models to calculate the probability of AI generation for both text and images.
* **Multi-Step Progress Tracking**: A clean, modern UI provides a clear streaming output of the agentic process (Extracting → Searching → Verifying) for maximum transparency.
* **Interactive Results**: Provides a detailed breakdown of "Verified True," "Partially True," and "Disproven" claims with linked external sources.
* **PDF Report Export**: Allows users to generate and download professional verification reports for offline sharing.
* **Persistence & Analytics**: Tracks live verification statistics and stores a comprehensive history of fact-checks using a serverless PostgreSQL backend.
* **Granular Accuracy Reports**: Generates detailed, interactive reports complete with explicit citations, confidence scores, and justifications for the AI's reasoning.
* **AI-Generated Text Detection**: Implements a forensics mechanism to analyze input text and calculate a probability score indicating if the content was likely generated by an LLM.
* **AI-Generated Media Detection**: Extends detection capabilities to analyze associated images to determine if they are synthetically manipulated or deepfakes.

## 🛠️ Tech Stack

* **Framework**: Next.js 15 (App Router).
* **Frontend**: React 19, Tailwind CSS, Lucide React.
* **Animations**: Motion (formerly Framer Motion).
* **AI Engine**: Google Gemini API via `@google/genai` and LangChain.
* **Database**: Neon (Serverless Postgres) with Drizzle ORM.
* **Export Tools**: jsPDF, html2canvas, and html-to-image.

## ⚙️ Getting Started

### Prerequisites
* **Node.js**: Ensure you have the latest LTS version installed.
* **Neon Database**: A serverless PostgreSQL instance from [Neon](https://neon.tech).
* **Google Gemini API Key**: Obtain an API key from [Google AI Studio](https://aistudio.google.com/).

### Installation

1.  **Clone the repository**:
    ```bash
    git clone <your-repo-url>
    cd BokshiGFG
    ```

2.  **Install dependencies**:
    ```bash
    npm install
    ```
    *(Note: This installs essential packages including Next.js 15, React 19, Drizzle ORM, and the Google Generative AI SDK)*

3.  **Environment Setup**:
    Create a `.env.local` file in the root directory and add your credentials:
    ```env
    DATABASE_URL=your_neon_postgres_url
    GEMINI_API_KEY=your_google_gemini_api_key
    ```
    *(Note: The `DATABASE_URL` is required by the Neon client and Drizzle configuration)*

4.  **Database Migration**:
    Initialize your database schema by pushing the definitions found in `lib/schema.ts` to your Neon instance:
    ```bash
    npx drizzle-kit push
    ```

5.  **Run Development Server**:
    Start the app locally:
    ```bash
    npm run dev
    ```
    The application will be available at http://localhost:3000.
